# 《替你说话》MVP 快速启动说明

## 1. 运行环境

- JDK 17+
- Maven 3.9+
- MySQL 8.x
- Redis 7.x
- （可选）PostgreSQL + pgvector（当前后端已包含配置）

## 2. 初始化数据库

1. 在 MySQL 中创建数据库：

```sql
CREATE DATABASE speak_for_you CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

2. 启动项目时会自动执行 `src/main/resources/schema.sql` 初始化表与基础数据。

## 3. 配置文件

编辑 `src/main/resources/application.yml`：

- `spring.datasource`：MySQL 连接信息
- `spring.data.redis`：Redis 连接信息
- `app.jwt.secret`：JWT 密钥（至少 32 位）
- `app.dashscope.system-api-key`：可选，系统通义千问 Key（为空则使用本地兜底回复）

## 4. 启动项目

在项目根目录执行：

```bash
mvn spring-boot:run
```

默认地址：`http://localhost:8080`

## 5. 前端入口与功能

- 登录页：`http://localhost:8080/index.html`
- 主应用：`http://localhost:8080/app.html`

MVP 已实现页面：

- 快速回复（SSE 流式，3 条回复）
- 对话模式（STOMP over WebSocket）
- 设置页（昵称、默认人格、API Key、模型、自定义人格 CRUD、今日用量）

## 6. STOMP 联调约定

前端 STOMP 连接：

- Endpoint：`ws://localhost:8080/ws/chat`
- CONNECT Header：`Authorization: Bearer <JWT>`

消息路由：

- 客户端发送：`/app/chat.send`
- 客户端订阅：`/user/queue/chat/{sessionId}`
- 发送体：

```json
{
  "sessionId": 100,
  "message": "客户说要延期付款，我该怎么回？"
}
```

服务端推送体：

```json
{ "type": "token", "data": "..." }
{ "type": "complete", "data": "完整建议回复" }
{ "type": "error", "data": "错误信息" }
```

## 7. 常见排查

- 登录失败：确认预置账号密码是否在 `schema.sql` 中已初始化。
- 页面空白/跳回登录：确认 JWT 是否成功写入浏览器 `localStorage`。
- STOMP 连接失败：检查 `Authorization` 头是否带 `Bearer` 前缀、JWT 是否过期。
- 快速回复失败：检查 Redis 是否可用（限流依赖 Redis）。
