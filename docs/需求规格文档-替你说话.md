# 《替你说话》需求规格文档（PRD）

> 版本：v1.0  
> 日期：2026-03-05  
> 状态：MVP 最小可行性产品

---

## 一、项目概述

### 1.1 项目名称

**替你说话**（SpeakForYou）

### 1.2 项目定位

一款面向**职场社恐人群**的 AI 智能聊天回复助手。用户复制他人发来的消息粘贴到系统中，系统根据预设的**人格风格**和**职场场景**，生成多条高质量回复供用户选择复制使用。

### 1.3 目标用户

- 职场中不善言辞、容易社交焦虑的人群
- 需要快速得体回复领导/客户/同事消息的职场人
- 希望提升沟通情商但不知如何措辞的人

### 1.4 核心价值

| 痛点 | 解决方案 |
|------|---------|
| 收到消息不知道怎么回 | AI 即时生成 3 条风格各异的回复 |
| 担心回复不得体 | 预设职场人格，确保措辞恰当 |
| 不同场景措辞拿捏不准 | 场景化预设，精准匹配沟通语境 |
| 需要更深入的沟通辅助 | 对话模式支持多轮交流，带上下文记忆 |



## 二、功能需求

### 2.1 功能架构图

```
替你说话
├── 用户系统
│   ├── 注册/登录（Spring Security）
│   ├── 个人设置
│   └── API Key 管理
├── 快速回复模式
│   ├── 粘贴消息输入
│   ├── 选择人格风格
│   ├── 选择场景
│   ├── RAG 话术检索 + AI 生成
│   └── 3 条回复展示 & 一键复制
├── 对话模式
│   ├── 多轮对话界面
│   ├── Prompt 驱动 + 上下文记忆
│   ├── 人格风格切换
│   └── 会话管理
├── 人格系统
│   ├── 系统预设人格（5种）
│   └── 用户自定义人格
├── 场景系统
│   └── 系统预设场景（4种）
└── 系统管理
    ├── 限流控制
    └── 话术知识库管理
```

### 2.2 核心功能详述

#### 2.2.1 用户系统

| 功能项 | 描述 |
|--------|------|
| 注册 | 不支持注册，后台自己添加用户 |
| 登录 | Spring Security 实现|
| 个人设置 | 修改昵称、可以选择默认人格，或者自己设置人格，场景只会在ai回复3条内容的时候，带上场景参数，会话模式是没有场景的 |
| API Key 管理 | 用户可填入自己的通义千问 API Key |
| 权限控制 | 未登录不可使用任何功能 |

#### 2.2.2 快速回复模式（核心功能）

**用户流程：**

```
# 第一种模式
用户登录 → 初始化自己的人设  →
用户复制他人消息 → 粘贴到输入框  → 选择场景
                                                    ↓
                              系统从话术知识库 RAG 检索 + AI 生成
                                                    ↓
                                    展示 3 条回复 → 用户选择 → 一键复制


```

**功能细节：**
- 输入方式：复制粘贴文本（MVP 阶段）
- 每次生成 **3 条**不同风格的回复
- 每条回复旁有"复制"按钮，一键复制到剪贴板
- 回复生成使用流式输出（Streaming），提升体验

#### 2.2.3 对话模式

**用户流程：**

```
用户登录 → 初始化自己的人设  →
进入会话 → 粘贴对方消息 → AI 基于人格+上下文生成回复建议
                                        ↓
                        用户可继续粘贴后续消息 → AI 记住上下文持续辅助 → 用户长按可以复制回复信息

                        
```

**功能细节：**
- 支持多轮对话，AI 记住当前会话的上下文
- 上下文窗口：保留当前会话内的所有对话（限制 50条）
- 每轮对话直接给出最匹配的回复语句，该回复会作为"你的回复"进入上下文
- 支持创建多个会话，每个会话独立记忆
- 使用 Prompt 驱动（非 RAG），通过系统提示词控制人格

#### 2.2.4 人格系统

**系统预设人格（5种，面向职场）：**

| 人格名称 | 风格描述 | 典型用语特征 |
|---------|---------|-------------|
| 专业得体 | 措辞严谨、礼貌周到、表达清晰 | "感谢您的反馈，我会尽快处理" |
| 轻松友好 | 亲切随意、带点小幽默 | "收到~ 这个我来搞定，放心！" |
| 委婉拒绝 | 圆滑婉转、不伤感情 | "最近确实忙不过来，要不咱看看其他方案？" |
| 高情商化解 | 化解矛盾、缓和气氛 | "理解你的想法，咱们一起看看怎么解决" |
| 简洁高效 | 言简意赅、直奔主题 | "好的，今天下班前给你" |

**用户自定义人格：**
- 用户可创建自己的人格，填写：人格名称、风格描述、说话语气特点
- 自定义人格保存在用户账号下
- 支持编辑和删除自定义人格

#### 2.2.5 场景系统

**系统预设场景（4种）：**

| 场景名称 | 适用情境 | AI 理解上下文侧重点 |
|---------|---------|-------------------|
| 同事日常聊天 | 和平级同事的日常交流 | 亲和力、话题延续、适当幽默 |
| 领导/上级沟通 | 和上司、领导的对话 | 尊重、专业、适度表态 |
| 客户/合作方沟通 | 商务往来、客户对接 | 礼貌、专业、利益平衡 |
| 委婉拒绝/推脱 | 需要婉拒的各类场景 | 给足面子、提供替代方案 |

**人格 + 场景组合使用：**
- 用户同时选择一个人格和一个场景
- AI 会综合人格风格和场景特征生成回复
- 例如：「轻松友好」人格 +「领导沟通」场景 = 既亲切又不失分寸

### 2.3 功能优先级

| 优先级 | 功能 | MVP 是否包含 |
|--------|------|-------------|

## 三、非功能需求



### 3.1 安全要求

| 要求 | 实现方式 |
|------|---------|
| 用户认证 | Spring Security |
| 密码存储 | BCrypt 加密 |
| API Key 存储 |
| 接口防护 | CSRF 防护 + 接口限流 |
| 聊天内容 | 不持久化用户粘贴的聊天内容（快速回复模式） |

### 3.3 限流策略

| 用户类型 | 限制 | 说明 |
|---------|------|------|
| 未登录用户 | 不可使用 | 必须登录 |
| 登录用户（系统 Key） | 30 次/天 | 使用系统默认 API Key |
| 登录用户（自带 Key） | 不限制 | 消耗用户自己的 API 额度 |

限流实现：Redis 计数器，key 为 `rate_limit:{userId}:{date}`，每次请求递增，超过阈值返回提示。

## 四、技术架构

### 4.1 技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 后端框架 | Spring Boot 3.x | Java 17+ |
| 安全框架 | Spring Security | 认证授权 |
| AI 框架 | LangChain4j | 对接通义千问、RAG、记忆管理 |
| 主数据库 | MySQL 8.x | 用户、人格、场景、配置数据 |
| 缓存/限流 | Redis | 会话缓存、限流计数、对话上下文临时存储 |
| 向量数据库 | PostgreSQL + pgvector | 话术知识库的向量存储与检索 |
| 前端 | HTML + CSS + JavaScript | Spring Boot 静态资源目录，不分离 |
| UI 框架 | Tailwind CSS + 自定义样式 | 温馨可爱风格 |
| AI 模型 | 通义千问（Qwen） | 支持 turbo/plus/max 可选 |

## 十二、风险与注意事项

| 风险 | 应对措施 |
|------|---------|
| AI 生成不当内容 | 在 Prompt 中加入内容安全约束 |
| 系统 API Key 被滥用 | 严格限流 + 登录才可用 |
系统默认使用 `qwen-plus`，用户可在设置中自行切换。
