# SpeakForYou MVP 接口文档（前端联调版）

## 1. 基础约定

- Base URL：`/api`
- 认证：JWT（除登录外都需要）
- 请求头：`Authorization: Bearer <token>`
- 非流式统一响应：
  - 成功：`{"code":0,"message":"ok","data":...}`
  - 失败：`{"code":非0,"message":"错误信息","data":null}`
- 时间格式：`yyyy-MM-dd HH:mm:ss`
- MVP 账号：不开放注册，账号由后台预置

## 2. 认证接口

### 2.1 登录

- `POST /auth/login`
- 请求体：

```json
{
  "username": "alice",
  "password": "123456"
}
```

- 响应 data：

```json
{
  "token": "jwt-token",
  "user": {
    "id": 1,
    "username": "alice",
    "nickname": "小A",
    "defaultPersonaId": 1,
    "modelName": "qwen-plus",
    "hasApiKey": false
  }
}
```

### 2.2 登出

- `POST /auth/logout`
- 说明：服务端将当前 token 加入黑名单
- 响应 data：`{"success": true}`

### 2.3 当前用户

- `GET /auth/me`
- 响应 data：同登录返回中的 `user`

## 3. 用户设置接口

### 3.1 更新个人信息

- `PUT /user/profile`
- 请求体：

```json
{
  "nickname": "小A",
  "defaultPersonaId": 2
}
```

### 3.2 设置自定义 API Key 与模型

- `PUT /user/api-key`
- 请求体：

```json
{
  "apiKey": "sk-xxxx",
  "modelName": "qwen-plus"
}
```

- `modelName` 可选：`qwen-turbo | qwen-plus | qwen-max`

### 3.3 清除自定义 API Key

- `DELETE /user/api-key`

### 3.4 查询今日用量

- `GET /user/usage`
- 响应 data：

```json
{
  "date": "2026-03-05",
  "used": 12,
  "limit": 30,
  "unlimited": false
}
```

说明：`unlimited=true` 代表用户已配置自己的 API Key，不受 30 次/天限制。

## 4. 人格与场景接口

### 4.1 获取人格列表（系统 + 我的）

- `GET /personas`
- 响应 data 示例：

```json
[
  {
    "id": 1,
    "name": "专业得体",
    "description": "措辞严谨、礼貌周到、表达清晰",
    "tone": "用\"您\"、结构化表达",
    "isSystem": true
  }
]
```

### 4.2 创建自定义人格

- `POST /personas`
- 请求体：

```json
{
  "name": "我的风格",
  "description": "简短但有温度",
  "tone": "礼貌、直接"
}
```

### 4.3 编辑自定义人格

- `PUT /personas/{id}`

### 4.4 删除自定义人格

- `DELETE /personas/{id}`

### 4.5 获取场景列表（系统预设）

- `GET /scenes`
- 响应 data 示例：

```json
[
  {
    "id": 1,
    "name": "同事日常聊天",
    "description": "和平级同事的日常交流"
  }
]
```

## 5. 快速回复接口（核心，SSE 流式）

### 5.1 流式生成 3 条回复

- `POST /quick-reply/stream`
- Headers：
  - `Authorization: Bearer <token>`
  - `Accept: text/event-stream`
- 请求体：

```json
{
  "message": "领导说这个方案今晚要给结果",
  "personaId": 1,
  "sceneId": 2
}
```

- 响应类型：`text/event-stream`
- 前端处理：
  - 按 token 逐步渲染
  - 流结束后将完整文本解析为 JSON 数组，展示 3 张回复卡片

模型目标输出（完整结束后）：

```json
[
  { "id": 1, "reply": "....", "style": "偏正式" },
  { "id": 2, "reply": "....", "style": "简洁直接" },
  { "id": 3, "reply": "....", "style": "高情商缓冲" }
]
```

## 6. 对话模式接口（会话管理 + WebSocket）

### 6.1 会话管理 REST

- `GET /chat/sessions`：会话列表
- `POST /chat/sessions`：创建会话

```json
{
  "personaId": 1,
  "title": "和领导沟通"
}
```

- `DELETE /chat/sessions/{id}`：删除会话
- `GET /chat/sessions/{id}/messages`：历史消息（最多最近 50 条）

### 6.2 WebSocket 实时对话

- 连接地址：`ws://{host}/ws/chat?token=<JWT>`
- 客户端发送：

```json
{
  "sessionId": 100,
  "message": "客户说要延期付款，我该怎么回？"
}
```

- 服务端推送：

```json
{ "type": "token", "data": "..." }
{ "type": "complete", "data": "完整建议回复" }
{ "type": "error", "data": "错误信息" }
```

## 7. 错误码（MVP 最小集）

- `0`：成功
- `4001`：参数错误
- `4003`：未登录或 token 无效
- `4004`：资源不存在（人格、会话等）
- `4290`：今日额度已用完（30 次）
- `5000`：系统错误
- `5001`：AI 服务调用失败（含上游限流、超时）

## 8. 前端必须遵守的业务规则（MVP）

- 快速回复必须传：`message + personaId + sceneId`
- 对话模式不使用 `sceneId`，仅绑定 `personaId`
- 仅当用户未配置自定义 API Key 时才走 30 次/天限流
- 快速回复输入不落库；对话模式消息会持久化（并缓存，窗口 50 条）
